# WC3 Item System Fixes - March 17, 2026

## Summary of Critical Bug Fixes

### 🐛 BUG #1: Duplicate Field Exports Causing Empty Tooltips
**Problem**: Extended Tooltip (utub) appeared empty in World Editor despite being populated in database.

**Root Cause**: FIELD_MAP had duplicate entries mapping to same WC3 field code:
- `'tooltip_extended'` → `'utub'`  
- `'extended_tooltip'` → `'utub'` ❌ (duplicate)
- `'actively_used'` → `'iusa'`
- `'use_automatically'` → `'iusa'` ❌ (duplicate)
- `'dropped_on_death'` → `'idrp'`
- `'drops_on_death'` → `'idrp'` ❌ (duplicate)

**Impact**: Each field was written TWICE to .w3t file. The second write (with NULL/empty value) overwrote the first write (with actual data).

**Evidence**: Hex dump showed 6 occurrences of 'utub' (2x per item):
- First utub: Full 249-322 character strings ✅
- Second utub: Empty strings (0 bytes) ❌ → Overwrote first

**Fix**: Removed duplicate entries from `core/wc3_w3t_exporter.py` FIELD_MAP
- Lines 46, 73, 77: Removed 'extended_tooltip', 'use_automatically', 'drops_on_death'

**After Fix**: Only 3 'utub' occurrences (1 per item), all with full data ✅

## Technical Details

### Binary Format Issue
The WC3 Reforged v3 .w3t format processes modifications sequentially. When duplicate fields are written, the last value wins. This caused:

```
Write 1: utub = "[249 chars of tooltip data]"
Write 2: utub = "" (empty, from 'extended_tooltip' NULL column)
Result in WE: "" (empty) ❌
```

After fix:
```
Write 1: utub = "[249 chars of tooltip data]"
Result in WE: "[249 chars of tooltip data]" ✅
```

### Database Schema Complexity
The database has evolved with multiple aliases for the same WC3 fields:
- `tooltip_extended` vs `extended_tooltip`
- `actively_used` vs `use_automatically`
- `dropped_on_death` vs `drops_on_death`

The FIELD_MAP must use **ONLY ONE** column name per WC3 field code to prevent duplicate exports.
