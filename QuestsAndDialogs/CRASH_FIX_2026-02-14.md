# Quest System Crash Fix - February 14, 2026

## Problem Report
**User Issue**: "Unexpected error occurs randomly which crashes the game - not sure if related to this but crash occurred shortly after 'OnFarewellEnd'"

## Root Cause Analysis

The crash was caused by **unsafe timer callbacks and message queue operations** in QuestMaster.j:

### Critical Issues Identified:

1. **Timer Callback Safety (ShowDelayedQuestDiscovered)**
   - Timer expired but QuestData wasn't validated before use
   - No check if quest was discovered before displaying
   - Missing validation for empty titles
   - Memory cleanup wasn't using Table.has() check

2. **Message Queue Race Conditions (ShowNextQuestUpdate)**
   - No validation of queue head index bounds before reading
   - Messages weren't cleared after reading (potential memory issue)
   - Multiple timer callbacks could conflict

3. **Queue Overflow (EnqueueQuestUpdateMessage)**
   - No bounds validation before writing to queue
   - Tail index could theoretically go out of bounds

4. **Double-Completion Crashes (complete method)**
   - Quest could be completed multiple times if callbacks fired rapidly
   - No guard against duplicate state transitions

5. **String Concatenation Crashes**
   - Empty titles concatenated without validation
   - Multiple public API functions built messages without safety checks
   - No protection against empty quest data

## Implemented Fixes

### 1. ShowDelayedQuestDiscovered (Lines ~125-151)
**Before:**
```jass
local QuestData q = QuestDiscoveredTimerData[GetHandleId(t)]
if q != 0 then
    set msg = "|cffffcc00QUEST DISCOVERED|r\n" + q.title
```

**After:**
```jass
local integer tId = GetHandleId(t)
local QuestData q = QuestDiscoveredTimerData[tId]

// Safety: validate quest data exists and is discovered
if q != 0 and q.discovered and q.title != "" then
    set msg = "|cffffcc00QUEST DISCOVERED|r\n" + q.title
    set objectives = q.formatObjectivesList()
    
// Clean up timer data
if QuestDiscoveredTimerData.has(tId) then
    call QuestDiscoveredTimerData.remove(tId)
endif
```

**Protection**: Validates quest exists, is discovered, and has valid title before string operations. Proper cleanup prevents memory leaks.

### 2. ShowNextQuestUpdate (Lines ~142-175)
**Before:**
```jass
if QuestUpdateQueueSize <= 0 then
    return
endif
set msg = QuestUpdateQueue[QuestUpdateQueueHead]
```

**After:**
```jass
// Safety: validate queue state
if QuestUpdateQueueSize <= 0 or QuestUpdateQueueHead <= 0 or QuestUpdateQueueHead > QUEST_UPDATE_QUEUE_MAX then
    set QuestUpdateQueueActive = false
    return
endif

set msg = QuestUpdateQueue[QuestUpdateQueueHead]
set QuestUpdateQueue[QuestUpdateQueueHead] = "" // Clear after reading
```

**Protection**: Validates queue index is within bounds (1-32). Clears slot after reading to prevent memory issues.

### 3. EnqueueQuestUpdateMessage (Lines ~177-214)
**Before:**
```jass
set QuestUpdateQueueTail = QuestUpdateQueueTail + 1
if QuestUpdateQueueTail > QUEST_UPDATE_QUEUE_MAX then
    set QuestUpdateQueueTail = 1
endif
set QuestUpdateQueue[QuestUpdateQueueTail] = msg
```

**After:**
```jass
set QuestUpdateQueueTail = QuestUpdateQueueTail + 1
if QuestUpdateQueueTail > QUEST_UPDATE_QUEUE_MAX then
    set QuestUpdateQueueTail = 1
endif

// Safety: validate tail index before writing
if QuestUpdateQueueTail < 1 or QuestUpdateQueueTail > QUEST_UPDATE_QUEUE_MAX then
    return
endif

set QuestUpdateQueue[QuestUpdateQueueTail] = msg
```

**Protection**: Double-checks array bounds before write operation to prevent array overflow crashes.

### 4. complete Method (Lines ~1489-1501)
**Before:**
```jass
method complete takes nothing returns nothing
    set this.completed = true
    set this.active = false
```

**After:**
```jass
method complete takes nothing returns nothing
    // Safety: prevent double-completion
    if this.completed or this.state == QUEST_STATE_COMPLETE then
        return
    endif
    
    set this.completed = true
    set this.active = false
```

**Protection**: Prevents quest from being completed twice, which could trigger duplicate messages and reward awards.

### 5. accept Method (Lines ~1460-1483)
**Before:**
```jass
method accept takes nothing returns nothing
    set this.active = true
    call this.setDiscovered(true)
```

**After:**
```jass
method accept takes nothing returns nothing
    // Safety: prevent double-accept or accepting already active quests
    if this.active or this.completed then
        return
    endif
    
    // Safety: validate quest has required data
    if this.title == "" then
        return
    endif
    
    set this.active = true
```

**Protection**: Prevents duplicate quest acceptance and validates quest has valid data before creating timer.

### 6. showCompletedMessage Method (Lines ~1350-1373)
**Before:**
```jass
call ClearTextMessages()
call QuestMessageBJ(bj_FORCE_ALL_PLAYERS, bj_QUESTMESSAGE_COMPLETED, this.formatCompleteMessage())
```

**After:**
```jass
// Safety: validate title exists
if this.title == "" then
    return
endif

set msg = this.formatCompleteMessage()
if msg != "" then
    call ClearTextMessages()
    call QuestMessageBJ(bj_FORCE_ALL_PLAYERS, bj_QUESTMESSAGE_COMPLETED, msg)

```

**Protection**: Validates quest title and message before attempting to display, preventing string concatenation crashes.

## Technical Details

### Queue Bounds
- Queue size: 1-32 (QUEST_UPDATE_QUEUE_MAX = 32)
- Valid indices: 1 through 32 (JASS arrays are 0-indexed but we use 1-based)
- Head and Tail pointers wrap around with modulo arithmetic

### Timer Lifecycle
1. **Created in accept()**: Stores QuestData reference in Table by timer handle ID
2. **Expires after 5 seconds**: ShowDelayedQuestDiscovered called
3. **Callback executes**: Validates quest, displays message, removes from Table
4. **Timer destroyed**: Prevents memory leak

### String Safety
- JASS strings are never truly "null" - only empty ("")
- All string checks use `!= ""` instead of null checks
- String concatenation validated at every step
