## Public APIs / Interfaces
- Add `PatrolFollowSystems/Companions.j` over `FollowSystem.j` with:
  - `Companions_Add(unit companionUnit, string companionIcon, unit leader, integer mode)`
  - `Companions_Remove(unit companionUnit)`
  - `Companions_SetLeader(unit companionUnit, unit leader)`
  - `Companions_SetMode(unit companionUnit, integer mode)`
  - `Companions_Suspend(unit companionUnit)`
  - `Companions_Resume(unit companionUnit)`
  - mode constants `COMPANION_MODE_DEFEND`, `COMPANION_MODE_PASSIVE`, `COMPANION_MODE_HOLD`
- Keep `CameraControl_FastPanToTarget` public and add a dedicated “reapply current camera mode” path for mouse-wheel reset, reusing the stored mode/state instead of creating a second camera system.
- No quest schema change; `qAradion` stays on `QuestGiver`/`QuestMaster`, but `QuestMaster.awardRewards` must be aligned to the OLDGUI reward behavior.

## Implementation Changes
- Shared quest rewards:
  - Align `QuestMaster.awardRewards` with `OLDGUI/Quest System Complete Rewards.md`: XP to all player-owned heroes plus hero companions in `Companion_Group`, gold/arena to Player 1, reputation via `AddReputation`/`AddReputationLinked` whenever `faction` is set.
  - Add temporary debug traces around quest completion and reward award paths during development, then leave them behind a debug flag only.
- Companion foundation:
  - Move Aradion and Valeria off direct `FollowSystem_SetFollow` + `QuestGiver_AddCompanion` orchestration and onto `Companions.j`.
  - `Companions.j` owns leader assignment, defend/passive/hold behavior, suspend/resume during cinematics and rituals, and synchronization with the existing GUI companion groups/icons.
- Ranger Missing:
  - Match `Valeria First Encounter.md` timing: ESC negotiation prompt only becomes active after the intro sequence is fully over.
  - Hold Nazgrek facing on Valeria during the standoff/negotiation window.
  - Remove unnecessary Ghost-state recovery unless the OLDGUI flow explicitly requires it.
  - Move every scripted teleport/reposition to the fully black part of fade transitions; if a cinematic is already active, move units outside camera view first and then walk them in.
  - Change the final objective text to `Speak with Aradion The Farseer` and make Aradion selection at that state trigger the completion cinematic directly, without the extra dialog button path.
- Fading Sparks:
  - Keep the current rod timing, but on a successful finish make the essence drop and Mana Wraith death happen on the same event, matching OLDGUI.
- Rifts of Corruption:
  - Use the OLDGUI `Quest Rifts Corruption*.md` files as the exact parity baseline for start order, bark order, fail flow, and escort-home gating.
  - Replace teleport-to-offset ritual start with: detect any open rift in range/rect, order Aradion to walk into channel range and cast `A04Z`, keep Valeria under companion control with patrol paused.
  - Re-register all living rift units after every fail/reset/recreate, and keep both unit-in-range and rect-based detection so any of the three rifts can start first.
  - Mark the active rift closed before the next proximity scan can fire, kill/remove the rift immediately on success, and clear any stale trigger/runtime state tied to that rift.
  - Keep bark order strictly sequential with one queue: start `Aradion -> Valeria`, finish-one `Aradion -> Valeria`, all-closed `Aradion -> Valeria -> Aradion -> Valeria`.
  - Keep Aradion in a dedicated ritual state: reissue the spell if interrupted, keep the channel animation looping while active, stop only on success/fail/death.
  - On third rift completion, set the last requirement text to `Escort Aradion and Valeria to Aradion's place`, keep quest/icon state in-progress until both are escorted home, and do not allow the completion question mark early.
  - On ritual death/fail, stop the ritual immediately, route the survivor to the fallen companion first, delay teleport/respawn/home reset, then rebuild rifts and quest state for retry.
- Camera / zone / UI / item cleanup:
  - Fix the existing `CameraControl` mouse-wheel reset path so it reapplies the current stored mode immediately, the same way PageUp/PageDown already do.
  - In `ZoneEvent`, after `MoveStart`, call `CameraControl_UpdateTargetCache` and then `CameraControl_FastPanToTarget` for teleported interiors/subzones.
  - Verify the `Shadowmaw Cave` registration path from `ZonesCore` through `ZoneEvent.MoveStart`/`HandleZoneEnter` and fix the broken enter-rect flow there.
  - In `StatsUI`, preserve the explicit Shadowclaw name/icon whenever the displayed pet is `udg_Shadowclaw` or the Shadowclaw unit type, not only when generic pet tracking is valid.
  - Keep `TasQuestBoxLight_PotS` Zones button hidden as a regression check.
  - In `ItemLootSystem`, keep pickup/use cleanup and add a fallback destroy/invalid-item cleanup path so powerup hover text is removed even if the item vanishes before the normal handlers catch it.
  - In `WC3ItemManager`, replace the hardcoded `|cffffcc00Ability:|r` tooltip label color with the defined item-class color lookup for `Ability`.
